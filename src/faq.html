SiteBuilder FAQ

Frequently Asked Questions

Answers to common questions about SiteBuilder, how it fits into engineering workflows, and what teams can expect.

What is SiteBuilder? +
SiteBuilder is an AI-assisted content editing workflow that lets non-technical teammates propose website updates in natural language, while engineers keep Git history, reviews, and deployment control.
Do I need to know how to code to make changes? +
No. The goal is that editors describe changes in plain language. Engineers can still review the resulting diffs before anything is merged.
How do approvals and code reviews work? +
Changes are intended to flow through normal Git workflows (branches, pull requests, and reviews). Your team decides what gets merged and deployed.
Is the AI allowed to deploy to production? +
The intended model is “assist, don’t autonomously ship.” Teams keep control through reviews and CI. (Exact permissions depend on how you integrate it.)
Can we use this with an existing site? +
Yes—SiteBuilder is designed to integrate with a repository you already have. How much work is required depends on your stack and how you structure content.
Where can I learn more or contribute? +
The project is open source. Visit the GitHub repository to explore the codebase and contribute.
